# Payment Portfolio Test Cases

This repository contains an Excel-based portfolio of payment test cases.

## What This Covers
- Card payments (success, decline, validation)
- QR and link payments
- Refunds and voids
- Settlements and transaction history
- Basic authentication and configuration flows

## File
- `Payment_Portfolio_Test_Cases.xlsx`

## How to Use
1. Open the Excel file.
2. Review the test case list and filter by feature, priority, or type.
3. Use the preconditions, steps, and expected results to execute manual tests.

## Typical Columns
- `Test Case ID`
- `Title`
- `Module`
- `Preconditions`
- `Steps`
- `Expected Result`
- `Priority`
- `Type`
- `Status`

## Notes
- This is a portfolio artifact intended for showcasing test design and coverage.
- No application source code is included in this repository.
